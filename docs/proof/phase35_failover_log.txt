## KEY IMPLEMENTATION CHANGES

### 1. BrainService.kt - Fixed Critical Bug
**BEFORE**: GEMMA_REASONING and CORE_BRAIN were combined into single case
```kotlin
BrainModelRole.GEMMA_REASONING, BrainModelRole.CORE_BRAIN -> evaluateModel(gemmaBrainModel, request, routeDecision)
```

**AFTER**: Properly separated cases
```kotlin
BrainModelRole.GEMMA_REASONING -> evaluateModel(gemmaBrainModel, request, routeDecision)
BrainModelRole.CORE_BRAIN -> evaluateModel(effectiveCoreBrainModel, request, routeDecision)
```

**Impact**: This fix was critical because CORE_BRAIN role would incorrectly use the Gemma runtime even when failing over to Qwen models, breaking the failover chain.

### 2. FailoverDebugReceiver.kt - New Phase 35 Test Controls
Implemented comprehensive debug controls for Phase 35 device proof:

**Actions Available**:
- `ACTION_FORCE_UNAVAILABLE` - Force specific role unavailable for testing
- `ACTION_FORCE_AVAILABLE` - Force specific role available
- `ACTION_RESET_ALL` - Reset all Phase 35 controls
- `ACTION_CHECK_STATUS` - Check current model states
- `ACTION_PROBE_CURRENT_STATE` - Probe current model behavior

**Key Features**:
- Process-local state control
- State reset between tests
- Role-specific marker files in app-private storage
- Safe error handling and logging
- No exposure in release builds

### 3. ModelRuntimeLoader.kt - Manual Path Fallback
Added `manualPathForRole()` method for fallback mechanism when official resolver fails:

**Fallback Chain**:
1. Try `modelInstallService.getReadyModelPath()` (official resolver)
2. Try `manualPathForRole()` (app-private fallback)
3. Return `UnavailablePhoneLocalLlmEngine()`

**CORRECTION (2026-06-18)**: `manualPathForRole()` was removed from `ModelRuntimeLoader.kt` during later hardening. Readiness/fallback path resolution is now handled by `ModelInstallService.repairModelPath()` plus `ModelPathResolver`'s existing tiered resolution, both of which route every candidate path through `ModelInstallVerifier.verify()` (size + extension + SHA-if-specified) before granting readiness. This historical section is retained for record but is no longer an accurate description of current source.

**Supported Roles**:
- CORE_BRAIN → gemma-3n-E2B-it-int4.litertlm
- MULTILINGUAL_BACKUP → qwen2.5-1.5b-instruct-q4_k_m.gguf  
- LITE_FALLBACK → qwen2.5-0.5b-instruct-q4_k_m.gguf
